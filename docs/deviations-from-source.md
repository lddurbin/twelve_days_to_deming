# Deviations from Source

This log records **material departures** from Henry Neave's *12 Days to Deming* course where the site's text, structure, or content differs from the NZOQ PDF originals. The project's stated principle is verbatim transcription, so every material departure is a deliberate choice and is captured here.

## Why this log exists

Future contributors should be able to see at a glance where the site has diverged, and readers cross-referencing the NZOQ PDFs should have a single index of what's different. Without this log, each new deviation gets relitigated from scratch because prior precedent isn't surfaced.

## What counts

Record an entry when the change is **material**. Don't record mechanical changes.

**Material** (record):
- Whole sentences, paragraphs, or passages removed, added, or reworded.
- Pedagogical elements (activities, cross-references, call-outs) omitted or restructured.
- Interactive affordances substituted for static text in a way that changes what the reader does.
- Reorganisation that moves content across chapter or day boundaries.

**Mechanical** (don't record):
- Typography, whitespace, punctuation normalisation.
- Minor reflowing of paragraphs for web layout.
- Image extraction from scans.
- Clear typo corrections where Neave's intent is unambiguous.
- Hyperlinking existing cross-references (e.g. "see page 42" → an in-page anchor) without altering wording.

When in doubt, record it — readers and future contributors are better served by over-inclusion than by silent omission.

## Entry format

Each deviation is an `## YYYY-MM-DD — short title` section with the following fields:

- **What** — one-line description of the change.
- **Where** — site location (chapter, appendix, or surface area).
- **Source reference** — the corresponding location in Neave's PDF, where identifiable.
- **Why** — the reason for departing from the source.
- **Decided in** — link to the issue or PR comment where the decision was locked.
- **Landed in** — implementing PR(s) and/or commit(s), or *Pending — tracked in #NNN* if not yet merged.

Entries are listed newest-first.

## 2026-05-27 — Replace Optional Extras Part E PDF panel and *MR̄* ÷ 1.128 histograms with R-generated figures

- **What** — Replaced two cropped-PNG figures on Part E pages 66–67 with R/ggplot2 chunks: `part-e-p66-pdfs.png` (the four σ = 1 parent distributions arranged in a 2×2 grid) and `part-e-p67-xbar-histograms.png` (four histograms of the estimator *MR̄* ÷ 1.128 from simulated baselines of length 12). Both chunks are composed inline with `patchwork` rather than via the existing `pdf_family_plot()` helper because Neave's printed layout for both pages is a hand-arranged 2×2 with corner labels — not a regular comparable-scale family.
- **Where** — `content/appendix/optional-extras/05-part-e-normality.qmd` lines 137 and 145 (PNG references replaced with R chunks); `assets/images/optional-extras/part-e-p66-pdfs.png` and `assets/images/optional-extras/part-e-p67-xbar-histograms.png` deleted.
- **Source reference** — Neave Optional Extras Part E pages 66–67. The page-65 prose specifies "each being in a version with σ = 1 for ease of comparison with the initial normal version" for the parent distributions. The page-67 figure caption identifies the estimator as "*MR̄* ÷ 1.128" and notes the comparison axes "from 0.0 to 2.5 in every panel". Neave's printed page 66 shows the four PDFs in a 2×2 grid where each panel auto-scales to its own peak so the *shape diversity* is the point; the page-67 histograms are framed by a shared 0.0–2.5 x axis so the *spread similarity* is the point.
- **Why** — Inherits the rationale of epic [#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309) and the Optional Extras sub-epic [#315](https://github.com/lddurbin/twelve_days_to_deming/issues/315). SVG output is crisp at any zoom, adapts to dark mode via the CSS-filter mechanism from [#310](https://github.com/lddurbin/twelve_days_to_deming/issues/310), and the simulation is now reproducible from a fixed seed rather than a one-off PDF crop. Layout faithfulness took priority over consolidating the work behind a new helper: `pdf_family_plot()`'s overlay/stack layouts both assume comparable scales across panels, which is the opposite of what Neave does on page 66.
- **Parametrisations (all σ = 1)** —
  - **Normal**: `dnorm(x, mean = 0, sd = 1)`.
  - **Uniform**: half-width √3 around zero, so `dunif(x, min = -√3, max = √3)` — variance is `(2 √3)² / 12 = 1`.
  - **Triangular (unsymmetric)**: a right-triangle PDF with mode at the left edge. On `[a, b]` with mode at `a`, variance is `(b − a)² / 18`; setting variance = 1 gives `b − a = 3 √2`. With `a = 0`, `b = 3 √2 ≈ 4.243`, mean `= √2 ≈ 1.414`. The PDF reads `f(x) = 2 (b − x) / (b − a)²` on `[0, b]`. Neave does not give numeric `(min, mode, max)` for this distribution anywhere in the text — only a shape illustration on Part D page 52 — so the *shape* (right-triangle peaking on the left) is the load-bearing source-fidelity constraint; the specific endpoints are chosen so σ = 1 as the page-65 prose requires.
  - **Exponential**: `dexp(x, rate = 1)`, which automatically has σ = 1.
- **Simulation parameters (page-67 chunk)** — N = 200,000 baselines of length 12 per distribution, with `set.seed(358)` per chunk. Neave used 10,000,000. 200,000 is enough for the "strikingly similar" visual claim of page 67 to read obviously (peaks near 1.0 for the first three families; visibly wider with a small right shift for the exponential), runs in well under a second per family, and avoids dragging down per-render Quarto times. Larger N produces visually indistinguishable histograms on a printed page.
- **Implementation note — layout vs helper widening** — The task brief offered two options: widen `pdf_family_plot()` to support a 2×2 grid layout, or compose with `patchwork` directly in the chunk. Chose the patchwork-in-chunk route because (a) the helper is built around *comparable* scales across panels, but Neave's page-66 figure deliberately *un-scales* each panel so the shape — not the density value — is what stands out; (b) the page-66 corner labels (NORMAL top-left, UNIFORM top-right, TRIANGULAR bottom-left, EXPONENTIAL bottom-right) form a diagonal layout that is unique to this figure and would have added a wide surface area to a helper used elsewhere only in two stack/overlay contexts; (c) the histograms chunk has its own bespoke (auto-scaled y per panel, fixed x = 0–2.5, subtitle labels) — different enough from the page-66 PDFs chunk that no shared abstraction was justified.
- **Colour deviations** — None for the page-66 PDFs: curves render in `CHART_FG` (black) matching Neave's printed page. The page-67 histograms also use black outlines on a white fill — again matching Neave's printed page (he draws unfilled bars there, not the project's usual red fill). No new colour tokens introduced.
- **Decided in** — Issue [#358](https://github.com/lddurbin/twelve_days_to_deming/issues/358) under the [#315](https://github.com/lddurbin/twelve_days_to_deming/issues/315) sub-epic / [#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309) epic.

The helper uses the project's existing red-bar + blue-curve visual idiom from `histogram_with_pdf()`, draws the histogram on the density scale so the unit-area *N*(0, 1) overlay sits naturally on top with no further rescaling, and applies `run_chart_theme()` for axis treatment and dark-mode parity. `mu` defaults to NULL with pilot-sample estimation as a fallback, but the chunks here all pass the closed-form `mu` for reproducibility.
- **PDF (decorative) chunks** — The two parent-distribution PDFs are drawn as bespoke shape illustrations in the chunks (not via `pdf_family_plot()`), matching the structure of the Part E page-66 mini-panels in PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377). - **Reviewer-feedback compliance (PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377))** — The PDF chunks are fully deterministic and carry no seed. Random-draw chunks use `runif`, `rnorm`, `rexp` without the `stats::` prefix to match the rest of `R/functions/main-functions.R`. - **Implementation note — Seed defaults.** Each helper takes a `seed` argument (no `set.seed` at file scope; reproducibility is per-call). Defaults were picked by sweeping seeds 1–300 against the characteristic-behaviour assertions in `tests/testthat/test-funnel-rules.R` and visually previewing the resulting 50-drop control charts at `sd = 5, target = 30` (the Optional Extras Part A scale). Choices: Rule 1 = **7** (stable around target), Rule 2 = **2** (lag-1 autocorrelation ≈ −0.58, clean zig-zag), Rule 3 = **1** (second-half amplitude ≈ 2.7× first-half), Rule 4 = **42** (clear drift away from target with comparable first/second-half spreads). Callers can override `seed` to explore alternative runs.

Tests cover both the wrapper path and the direct-call path; both return ggplot objects.
- **Implementation note — Loop vs vectorisation.** Rules 1 and 4 are fully vectorisable (Rule 1 is plain `rnorm`; Rule 4 is `target + cumsum(noise)`) so they ship without a loop. Rule 2 also has a clean closed form — substituting `rest[i] = aim[i] + noise[i]` into the recurrence collapses to `rest[i] = target + noise[i] − noise[i−1]` for i ≥ 2 — but is implemented with a `for` loop here for parity with Rule 3 (whose alternating-cumulative-sum closed form reads less clearly than its recurrence) and because the chart consumer uses n ≤ 50, where no measurable benefit accrues. The doc-comment on each helper makes the loop-vs-closed-form choice explicit. This is consistent with PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377)'s reviewer feedback ("prefer vectorised forms over `apply()`; loops are fine when sequential dependencies make vectorisation impossible").
- **Implementation note — File organisation.** Put the four sim helpers in a new file `R/functions/funnel-rules.R` rather than appending to `R/functions/main-functions.R` (which is already ~1500 lines). Sourced from `R/setup.R` alongside the main file.
