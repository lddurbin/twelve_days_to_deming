## Executive summary

The project has landed v0.1.0 in excellent shape. Every quantitative regression flagged in the 2026-04-17 evaluation has been resolved: the 293 unlabelled OJS inputs are now zero (multi-line label coverage is 100 %, the previous count was a counting artefact + Days 7-12 still being mid-flight at the time), Day 12's H1-inside-`thought_commentary` violations are gone, Days 8/9's `major_activity_title` divs are now correctly wrapping `<h2>`, Day 7's 18 inline `style="color: blue"` spans are gone, and the "study notes you can't save" gap on Days 10-11 has been closed (76 → 96 `createDownloadButton` calls). All 116 QMD files render cleanly via `quarto render` (zero warnings, exit 0), all 12 day manifests pass `check-structure.sh` (484 of 484 checks green), the JS test suite is 55/55, R unit tests are 33/33, and the deploy and accessibility CI workflows are both wired and pinned to SHA-locked actions.

The drift surface for v0.1.0 maintenance is therefore narrow but real, and falls into four buckets: **(1) documentation that hasn't kept up with the shipped patterns** — `workflow/PATTERNS.md` still illustrates the obsolete `downloadNotes(...)` import in its Activity template and uses `# H1` syntax for `.thought` / `.thought_commentary` blocks even though every shipped chapter uses `## H2`; the unconverted `assets/templates/activity-template.qmd` is the only remaining live `downloadNotes` import in user-authored space. **(2) ARIA drift in the latest-converted chapters** — 10 of 35 `thought_commentary` divs (all in Days 10-12) ship without the `role="note" aria-label="Commentary"` attributes that all 25 of the earlier instances carry. **(3) Validation coverage gaps** — `check-structure.sh` only covers 12/13 manifestable content blocks (the `contributions-balaji-reddie` appendix has a manifest; the main appendix and Optional Extras have briefs but no manifests, so their structural drift is not policed by CI). The pa11y job only audits 11 day-chapter URLs (all from Days 1-3) plus 14 appendix pages — the entire Day 4-12 prose surface ships without per-page automated WCAG checks. **(4) Housekeeping** — `_quarto.yml.backup` (a stale Sept 2025 13-day-pre-v0.1.0 snapshot) is tracked in git and indistinguishable from the live config to a casual reader.

None of these are P0. There are no broken references (all 116 QMD files cross-link cleanly; all `#sec-pageNN` anchors resolve; all 7 `<dfn id="...">` IDs map to glossary anchors). There are zero broken image references. The renv.lock is in good shape for CI but mildly out of sync with the local dev installation. **Overall score: 4.6 / 5.0**, up 0.2 on the 2026-04-17 evaluation, with the improvement carried by Content (regressions cleared), Build Health (CI tightened, structure check now runs in CI per PR #276), and Accessibility (label coverage corrected, dyslexia font + dark mode + glossary + long-descriptions all shipped).

## 1. Drift detection across the 12 released days (4/5)

### Strengths
- **Download-button pattern is uniformly the new pattern.** Across the 88 day chapters, `createDownloadButton` is used 96 times and the legacy `downloadNotes(...)` import is used **zero** times. The only `downloadNotes` reference in the source tree is inside `assets/scripts/functions.js` itself (where `createDownloadButton` calls it as a private helper) and inside `assets/templates/activity-template.qmd` (a template file not wired into `_quarto.yml`).
- **`major_activity_title` is consistent.** All 7 instances either wrap an explicit `<h2>` or use Quarto-rendered `## H2` syntax — no plain-text or H1 variants remain.
- **CSS class usage is convergent.** Every project-specific class used in `content/` (`thought`, `thought_commentary`, `deming_quote`, `neave_note`, `principle_callout`, `return_callout`, `aside_callout`, `callout-emphasis`, `foreman-remark`, `separator*`, `spacer-*`, `mt-*`, `float-box*`, `technical_aid`, `activity_afterthought`, `major_activity_title`, `aside_callout`, `fe-*`, `data-sequence-mono`, `text-highlight-red`, etc.) resolves to a definition in `assets/styles/main.css`. No undefined project classes were found.
- **Inline styles are down to 10 site-wide** (from 28 at the previous evaluation, with 1.5× more files), distributed: Day 1 = 1, Day 2 = 3, Day 3 = 2, Day 6 = 1, Day 8 = 2, Day 12 = 1. All are justifiable one-offs (image float, column wrap, OJS html`...` literals).
- **R helpers are all live.** Each of the 6 functions in `R/functions/main-functions.R` is referenced from at least 2 files; `run_chart_theme` is the lowest-traffic helper but it is correctly used by `run_chart_plot`.

### Findings
- **[P2] PATTERNS.md is stale on the download-button pattern.** Lines 161-179 of `workflow/PATTERNS.md` show the Activity-with-text-input + Download-button template using `import { downloadNotes } from "../../../assets/scripts/functions.js"` and a manual `download_button.onclick = () => { downloadNotes(...) }` wiring. Every shipped chapter uses `import { createDownloadButton }` and a single-call pattern. A new contributor copy-pasting from PATTERNS.md will produce a non-conforming chapter that still works (because `downloadNotes` is exported) but drifts from the house style and is harder to test.
- **[P2] PATTERNS.md is stale on heading levels for thought blocks.** Lines 106, 123, and 146 of PATTERNS.md show `# PAUSE FOR THOUGHT X-y` and `# ACTIVITY X-y` (H1) inside `.thought` / `.thought_commentary` divs. Every shipped chapter uses `## H2` — a deliberate fix made during Day 12 cleanup. The PATTERNS.md template still encodes the old (broken) pattern.
- **[P2] `assets/templates/activity-template.qmd` is also stale.** Same `downloadNotes` import + same H1 heading. Not wired into `_quarto.yml`, but anyone using it as a starting point inherits both anti-patterns. Either delete it (templates aren't being used to scaffold new chapters during maintenance) or update it to match the live pattern.

### Recommendations
- **[P2-1, NEW]** Update `workflow/PATTERNS.md` to show the `createDownloadButton` pattern and `## H2` heading inside `.thought` / `.thought_commentary` blocks. Reference the canonical implementations in Day 1, chapter 13 (large download button) and Day 6, chapter 3 (`thought` with H2). **Quick win.**
- **[P2-2, NEW]** Either delete `assets/templates/activity-template.qmd` or rewrite it against the shipped pattern. **Quick win.**
