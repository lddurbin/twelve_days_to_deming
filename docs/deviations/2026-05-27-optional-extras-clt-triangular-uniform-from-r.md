## 2026-05-27 — Replace Optional Extras Part D CLT-demo triangular and uniform PNGs with R-generated figures

- **What** — Replaced eight cropped-PNG figures on Optional Extras Part D pages 52–53 with R/ggplot2 chunks: the triangular and uniform parent-distribution PDFs, and six histograms of the standardised sample mean *Z* = (*X̄* − μ) / (σ / √*n*) at *n* = 2, 4, and 10 for each parent. The exponential family on pages 53–54 (`part-d-p53-exp-pdf.png`, `part-d-p53-exp-n2.png`, `part-d-p53-exp-n4.png`, `part-d-p53-exp-n10.png`, `part-d-p54-exp-n100.png`) is out of scope for this PR — it is tracked under issue [#357](https://github.com/lddurbin/twelve_days_to_deming/issues/357) and will reuse the same `clt_demo_plot()` helper introduced here.
- **Where** — `content/appendix/optional-extras/04-part-d-crash-course.qmd` (eight inline-image references replaced by R chunks; the orphan PNGs in `assets/images/optional-extras/` remain on disk per the epic-[#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309) policy and will be removed in a follow-up PR after production verification).
- **Source reference** — Neave Optional Extras Part D pages 52–53. The page-51 prose introduces the simulation context ("samples were generated 10,000,000 times"); pages 52–53 show eight figure captions verbatim reproduced as `fig-cap` strings in the new chunks. Neave's parametrisations of the parent distributions are not stated numerically — the figures are shape illustrations — so the choices below derive from the σ = 1 constraint his prose specifies (and which Part E pages 65–67 then explicitly reuse "for ease of comparison with the initial normal version").
- **Why** — Inherits the rationale of epic [#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309) and the Optional Extras sub-epic [#315](https://github.com/lddurbin/twelve_days_to_deming/issues/315). SVG output is crisp at any zoom, adapts to dark mode via the CSS-filter mechanism from [#310](https://github.com/lddurbin/twelve_days_to_deming/issues/310), and the simulation is now reproducible from a fixed seed.
- **Parametrisations (all σ = 1)** —
  - **Uniform**: half-width √3 around zero, so `runif(n, min = -√3, max = √3)`. Variance is `(2 √3)² / 12 = 1`. Same parametrisation as PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377) Part E page 66.
  - **Triangular (unsymmetric, right-triangle decreasing from left peak)**: on `[0, b]` with mode at `a = 0`, variance is `(b − a)² / 18`; setting variance = 1 gives `b = 3 √2 ≈ 4.243`, mean `= b / 3 = √2 ≈ 1.414`. The PDF reads `f(x) = 2 (b − x) / b²` on `[0, b]`. Inverse-CDF sampling: `x = b * (1 - sqrt(1 - U))` for `U ~ Uniform(0, 1)`. Same parametrisation as PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377).
- **Simulation parameters** — `n_samples = 100000` standardised *X̄* values per histogram (helper default), with `set.seed(356)` at the top of each random-draw chunk. Neave used 10,000,000; 100,000 is comfortably enough to make the CLT shape claims visually obvious on a printed/screen page and keeps per-chunk render time well under a second. Larger N produces visually indistinguishable histograms at the printed resolution.
- **New helper — `clt_demo_plot(parent_dist, n, …)`** — Added to `R/functions/main-functions.R` next to `histogram_with_pdf()`. Takes a parent-distribution *sampler* (a `function(n) numeric`) plus a sample size, simulates the standardised sample mean *Z*, and returns a histogram with the standard normal *N*(0, 1) density overlaid. Designed deliberately to accept *any* sampler so issue [#357](https://github.com/lddurbin/twelve_days_to_deming/issues/357) can reuse it via `function(n) rexp(n, rate = 1)` for the exponential family on pages 53–54. The helper uses the project's existing red-bar + blue-curve visual idiom from `histogram_with_pdf()`, draws the histogram on the density scale so the unit-area *N*(0, 1) overlay sits naturally on top with no further rescaling, and applies `run_chart_theme()` for axis treatment and dark-mode parity. `mu` defaults to NULL with pilot-sample estimation as a fallback, but the chunks here all pass the closed-form `mu` for reproducibility.
- **PDF (decorative) chunks** — The two parent-distribution PDFs are drawn as bespoke shape illustrations in the chunks (not via `pdf_family_plot()`), matching the structure of the Part E page-66 mini-panels in PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377). They render as thin black outlines over a horizontal baseline with no axis ink — Neave's printed page has no axes — and are deterministic, so they carry no `set.seed()` call.
- **Reviewer-feedback compliance (PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377))** — The PDF chunks are fully deterministic and carry no seed. Random-draw chunks use `runif`, `rnorm`, `rexp` without the `stats::` prefix to match the rest of `R/functions/main-functions.R`. No `apply()` in the helper — `rowMeans()` is used for the row-wise mean, the only reduction needed.
- **Colour deviations** — None. Histogram bars use `CHART_LINE_COLOUR` (red) and the *N*(0, 1) overlay uses `CONTROL_LIMIT_COLOUR` (blue), both already documented dark-mode-safe tokens in the conventions block at the top of `R/functions/main-functions.R`. No new colour tokens introduced.
