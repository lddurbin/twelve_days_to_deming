## 2026-05-27 — Day 12 radar diagram rendered from R; small/key-table kept as PNG

- **What** — Replaced the cropped-PNG figure `assets/images/day-12/radar-diagram-large.png` on Day 12 page 3 (Neave's blank four-arm "kite" diagram for Activity 12–a Part 3) with an R/ggplot2 chunk driven by a new `radar_diagram_plot()` helper. The companion assets `radar-diagram-small.png` (printable worked-example template) and `radar-diagram-key-table.png` (the 14 Points + 5 Deadly Diseases KEY table) stay as PNGs in this PR — see *Why* below for the keep-as-PNG rationale on each.
- **Where** — `R/functions/main-functions.R` (added `radar_diagram_plot()` immediately after `specification_limits_diagram()`); `content/days/day-12/02-your-radar-diagram.qmd` (the `radar-diagram-large.png` Markdown image reference replaced by an R chunk); `tests/testthat/test-radar-diagram.R` (three new tests); `workflow/validation/day-12-manifest.yml` (figure list trimmed to drop the large-radar entry).
- **Source reference** — Neave Day 12 page 3 ("This diagram is used in Activity 12–a, Part 3 and is also on Workbook page 217"). The reference layout has four labelled arms meeting at a common origin: PSYCHOLOGY (top), SYSTEM (right), KNOWLEDGE (bottom), VARIATION (left). Each arm carries six labels reading "0" at the outermost tip down to "5" closest to the origin, with five tick marks dividing the arm into evenly-spaced intervals.
- **Why** — Inherits the rationale of epic [#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309): SVG output is crisp at any zoom, adapts to dark mode via the CSS-filter mechanism from [#310](https://github.com/lddurbin/twelve_days_to_deming/issues/310), and the radar's geometry now lives in code where any future correction (an axis label, an arm direction) is a one-line edit rather than a re-crop.
- **Implementation note — hand-rolled `coord_fixed()`, not `ggradar`** — The `ggradar` package renders closed-ring polygon radars with concentric circles and arms equally spaced around 360°. Neave's "radar" is in fact a Cartesian crosshair with four orthogonal arms and a 0-to-5 *linear* scale on each — closer to a pair of axes than to a polygon radar. Forcing `ggradar` would have required overriding most of its defaults (gridlines, ring fills, axis-label rotations) and would still leave the inner-most "5" labels misplaced relative to Neave's printed convention. A 100-line `coord_fixed()` helper using `annotate("segment", ...)` and `geom_text()` was both more faithful to the source layout and zero-dependency-cost (the project already imports `ggplot2`). Decision made during the 30-minute spike called for in [#360](https://github.com/lddurbin/twelve_days_to_deming/issues/360).
- **Implementation note — keep `radar-diagram-small.png` as PNG** — The small radar is a worked-example illustration showing Neave's Point 1 plotted with broken-green-dashes (4 plotted points + the polygon connecting them + a KEY-style dashed line specimen). Reproducing that in R would require additional helper plumbing for the legend-style dashed-line specimen and the example-score annotations, all to render a figure that is essentially explanatory rather than data-driven. The PNG is high-resolution and accessible (alt-text covered by `{.lightbox}`); converting it carries no editorial upside over keeping the source as-is. Recorded here so this PNG isn't flagged as an oversight in future audits. Per epic [#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309) the file is *not* deleted.
- **Implementation note — keep `radar-diagram-key-table.png` as PNG (for now)** — The 14 Points + 5 Deadly Diseases KEY table is fundamentally a *table* with a 19-row layout, a wide "KEY" column where learners record their colour/line-style choices, and a fixed font. The right tool for that conversion is `gt` (which the project already uses for the Red Beads tables) rather than ggplot2 — and the `gt` work is sized differently from the R-figures epic. Tracking the `gt` conversion as a separate effort keeps this PR scoped to the radar chart itself. Recorded here so the keep-as-PNG decision isn't mistaken for inaction.
