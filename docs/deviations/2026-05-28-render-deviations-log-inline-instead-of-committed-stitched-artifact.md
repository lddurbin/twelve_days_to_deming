## 2026-05-28 — Render deviations log inline at site-build time instead of committing a stitched artifact

- **What** — Removed the committed `docs/deviations-from-source.md` build artifact and the `scripts/build-deviations-log.R` script that produced it. The full deviations log is now rendered into `changes-from-source.qmd` at site-build time by an R chunk that calls a new `deviations_log_emit()` helper. Per-entry source files in `docs/deviations/` remain the only source of truth, unchanged.
- **Where** — Added `R/functions/deviations-log.R` (`deviations_log_assemble()` + `deviations_log_emit()`), sourced from `R/setup.R`. Added a `results: asis` R chunk to `changes-from-source.qmd` and rewrote the prose around it (the GitHub-URL link to the old stitched file is gone; readers now see the full log inline on the same page). Deleted `docs/deviations-from-source.md` and `scripts/build-deviations-log.R`. Added `docs/deviations-from-source.md` to `.gitignore` so a stale local build can't sneak it back. Updated `docs/deviations/README.md` to describe the new flow. Updated three rationale-pointing comments in `R/data/day-03-six-processes.R` and one bullet in `docs/glossary-audit.md` to point at `docs/deviations/` rather than the removed file.
- **Source reference** — n/a. This change is internal to the project's contribution workflow; nothing in Neave's source materials is affected.
- **Why** — The per-entry split in PR [#372](https://github.com/lddurbin/twelve_days_to_deming/pull/372) eliminated *editorial* conflicts on the deviations log but left a *textual* conflict on the regenerated stitched file. The Wave 1 concurrent-PR fleet ([#378](https://github.com/lddurbin/twelve_days_to_deming/pull/378)–[#383](https://github.com/lddurbin/twelve_days_to_deming/pull/383)) made the cost visible: 4 of 5 PRs paid the mechanical-regen tax on rebase, each forcing a CI re-run (~5 min pa11y long pole). Inlining the render eliminates both the build artifact and the regen step. Tracked in [#384](https://github.com/lddurbin/twelve_days_to_deming/issues/384).
- **Implementation note — chunk uses `results: asis`** — Quarto's R chunks default to wrapping printed output in a code block. `results: asis` tells knitr to emit `cat()` output as raw markdown that Quarto then processes — so per-entry headings, links, and bullets render as themselves rather than as a verbatim block.
- **Implementation note — heading downshift** — `deviations_log_emit()` downshifts per-entry `##` headings to `###` so they nest under the page's existing `## The full technical log` section instead of colliding with it. The per-entry files on disk are unchanged; the downshift happens in memory at render time.
- **Implementation note — script removed, not preserved as a thin wrapper** — The previous `scripts/build-deviations-log.R` existed only to write the now-removed stitched artifact. Maintainers who want to inspect the raw stitched markdown can `source("R/functions/deviations-log.R")` and call `deviations_log_assemble(downshift_headings = FALSE)` from an R session, as documented in the updated `docs/deviations/README.md`.
