## 2026-05-27 — Funnel-rule simulation helpers + chart wrapper (infrastructure)

- **What** — Added four R simulation helpers (`funnel_rule_1_sim()` … `funnel_rule_4_sim()`) that generate marble-resting-position sequences under Deming's four Funnel-Experiment rules, plus a `funnel_simulation_chart_plot()` wrapper that pipes a sequence through `run_chart_plot()` with MR-based control limits derived from a configurable baseline. Also added a scratch developer-facing demo qmd at `inst/demo/funnel-rules-demo.qmd` (not linked from `_quarto.yml`; will be deleted by the consumer PR).
- **Where** — `R/functions/funnel-rules.R` (new), sourced from `R/setup.R`; `tests/testthat/test-funnel-rules.R` (new — 22 test cases); `inst/demo/funnel-rules-demo.qmd` (new — scratch); `.gitignore` (ignores `inst/**/*_files/` and `inst/demo/*.html` so demo renders never enter the tree).
- **Source reference** — Neave Day 3: "Rule 1 of the Funnel (Ford's Second Strategy)" and "Rule 2 of the Funnel (Ford's First Strategy)" in `content/days/day-03/11-the-first-two-rules-of-the-funnel.qmd`; "Rules 3 and 4 of the Funnel" in `content/days/day-03/12-rules-3-and-4-of-the-funnel.qmd`. The four rules are also discussed in Neave's *The Deming Dimension* Chapter 5 and (with control-chart treatment) on Optional Extras Part A pages 9–14 (`content/appendix/optional-extras/01-part-a-funnel-charts.qmd`).
- **Why** — This is *pure infrastructure*. No learner-facing qmd page is modified in this PR. Issue [#352](https://github.com/lddurbin/twelve_days_to_deming/issues/352) was closed in [#373](https://github.com/lddurbin/twelve_days_to_deming/pull/373) as keep-as-PNG because re-plotting `assets/images/optional-extras/rule-1-short.png` and `rule-1-long.png` faithfully would require Neave's original Monte Carlo sequences (which are not in the repo or the source PDF), and the existing `funnel_track_plot()` helper draws the 20–40 marble-track strip, not an XmR control chart. That blocker had two halves: missing **simulation generators** for each rule, and a missing **chart wrapper** to pipe the resulting sequences through `run_chart_plot()`. This PR ships both halves so a follow-up consumer PR can swap the rule-1 (and, if the same precedent is reopened, rule-2/3/4) control-chart PNGs for R-generated SVGs that re-skin under dark mode via the [#310](https://github.com/lddurbin/twelve_days_to_deming/issues/310) CSS-filter mechanism. The resulting sequences are *illustrative*, not Neave's exact simulations — the deviations-log entry recording the eventual PNG→R swap will need to document that consciously (similar in shape to the page-15 dice-data "Data decision A" in [the 2026-05-26 charts-a1-a2-purple-limits entry](https://github.com/lddurbin/twelve_days_to_deming/blob/main/docs/deviations/2026-05-26-replace-day-3-charts-a1-a2-purple-limits-png-with-an-r-gener.md)).
- **Implementation note — Seed defaults.** Each helper takes a `seed` argument (no `set.seed` at file scope; reproducibility is per-call). Defaults were picked by sweeping seeds 1–300 against the characteristic-behaviour assertions in `tests/testthat/test-funnel-rules.R` and visually previewing the resulting 50-drop control charts at `sd = 5, target = 30` (the Optional Extras Part A scale). Choices: Rule 1 = **7** (stable around target), Rule 2 = **2** (lag-1 autocorrelation ≈ −0.58, clean zig-zag), Rule 3 = **1** (second-half amplitude ≈ 2.7× first-half), Rule 4 = **42** (clear drift away from target with comparable first/second-half spreads). Callers can override `seed` to explore alternative runs.
- **Implementation note — Chart wrapper vs direct `run_chart_plot()` call.** Chose to ship the `funnel_simulation_chart_plot()` wrapper *in addition to* documenting that callers may pipe simulation output straight into `run_chart_plot()`. The wrapper bundles the MR-baseline-then-extend recipe that the Optional Extras Part A figures all share, which keeps the consumer call site to one line per chart and makes the baseline policy uniform across all four rules. Tests cover both the wrapper path and the direct-call path; both return ggplot objects.
- **Implementation note — Loop vs vectorisation.** Rules 1 and 4 are fully vectorisable (Rule 1 is plain `rnorm`; Rule 4 is `target + cumsum(noise)`) so they ship without a loop. Rule 2 also has a clean closed form — substituting `rest[i] = aim[i] + noise[i]` into the recurrence collapses to `rest[i] = target + noise[i] − noise[i−1]` for i ≥ 2 — but is implemented with a `for` loop here for parity with Rule 3 (whose alternating-cumulative-sum closed form reads less clearly than its recurrence) and because the chart consumer uses n ≤ 50, where no measurable benefit accrues. The doc-comment on each helper makes the loop-vs-closed-form choice explicit. This is consistent with PR [#377](https://github.com/lddurbin/twelve_days_to_deming/pull/377)'s reviewer feedback ("prefer vectorised forms over `apply()`; loops are fine when sequential dependencies make vectorisation impossible").
- **Implementation note — File organisation.** Put the four sim helpers in a new file `R/functions/funnel-rules.R` rather than appending to `R/functions/main-functions.R` (which is already ~1500 lines). Sourced from `R/setup.R` alongside the main file.
