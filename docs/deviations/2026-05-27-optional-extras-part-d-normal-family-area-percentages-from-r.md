## 2026-05-27 — Replace Optional Extras Part D normal-family and area-percentages PNGs with R-generated figures

- **What** — Replaced the two cropped-PNG illustrations on Optional Extras printed pages 47 and 49 with R/ggplot2 chunks. The page-47 figure (three normals with the same μ and increasing σ) is rendered by a new reusable `pdf_family_plot()` helper, with σ values fixed at 1, 2 and 3 (the prose only contrasts σ qualitatively as "Smallish σ / Larger σ / Still larger σ", so any monotonic triple in roughly 1:2:3 ratio carries the same pedagogical point). The page-49 figure (the same normal curve annotated with the standard 68/95/99.7 area percentages) is implemented directly in the chunk with `geom_ribbon()` regions and labelled `geom_text()`s for 34.1%, 13.6%, 2.15% and 0.135%.
- **Where** — `R/functions/main-functions.R` (added `pdf_family_plot()` directly after `histogram_with_pdf()`); `content/appendix/optional-extras/04-part-d-crash-course.qmd` (two `![…](png){.lightbox}` references replaced with R chunks carrying `fig-cap`/`fig-alt`). The orphan PNGs `assets/images/optional-extras/part-d-p47-normal-family.png` and `assets/images/optional-extras/part-d-p49-area-percentages.png` are intentionally left in place per the epic-#309 policy (delete only after the conversion has been verified in production).
- **Source reference** — Neave Optional Extras printed pages 47 and 49 (PDF pages 53 and 55), section "Continuous probability distributions, particularly the normal distribution". The accompanying prose on page 46 introduces the family-of-normals idea ("if σ is small then the distribution is tall and thin … if σ is large then the distribution is wider and flatter") and the page-49 figure is introduced on page 48 ("on the next page there are the same three 'graphs of the normal curve' as above but now with some 'proportions of area thereunder' inserted").
- **Why** — Inherits the rationale of epic [#309](https://github.com/lddurbin/twelve_days_to_deming/issues/309): SVG output is crisp at any zoom, adapts to dark mode via the CSS-filter mechanism from [#310](https://github.com/lddurbin/twelve_days_to_deming/issues/310), and turns chart edits into one-line code changes. This is the second ticket in the Optional Extras sub-epic [#315](https://github.com/lddurbin/twelve_days_to_deming/issues/315) and introduces the `pdf_family_plot()` helper that [#358](https://github.com/lddurbin/twelve_days_to_deming/issues/358) (Part E PDFs + x-bar histograms) will reuse to compare four different distribution families (normal / uniform / triangular / exponential).
- **Layout deviation — overlay vs three stacked panels (page 47)** — Neave's printed page-47 figure stacks the three curves *vertically* as three separate panels, each labelled "Smallish σ", "Larger σ" or "Still larger σ" with its own μ−3σ … μ+3σ x axis. The R replacement overlays the three curves on a *single* panel and tags each via a colour legend (red / blue / black, all dark-mode-safe project tokens). Overlay makes the σ progression directly visible (heights and widths comparable at a glance) rather than requiring the reader to mentally line up three independent panels; the underlying claim — same μ, varying σ shape only — is unchanged. The x-axis tick spacing on the overlay uses the *middle* curve's σ (σ = 2) for "μ−3σ … μ+3σ" so the labels read at a sensible density across the panel; this is a sacrifice of the per-panel "labels match each curve's own σ" property in Neave's stacked layout, but is the natural cost of the overlay rendering.
- **Layout deviation — single curve vs three stacked panels (page 49)** — Neave's printed page-49 figure repeats the same three curves from page 47, now each shaded with the standard percentages. Because the area percentages (34.1%, 13.6%, 2.15%, 0.135%) are *scale-invariant* — Neave's whole point on page 48 is that they "apply *irrespective of the particular values of μ and σ*" — the R replacement shows a single canonical curve with the percentages shaded once. This is materially less repetitive without weakening the pedagogical claim; the prose intro ("for *any* values of μ and σ … the probability of the normal random variable lying in the interval between μ−2σ and μ+2σ … is 2 × (34.1% + 13.6%) = 95.4%") is exactly what the single figure is illustrating.
- **Colour palette deviation (page 49)** — Neave's printed shaded regions use a yellow / magenta / green / red palette (centre / next band / next band / tails). The R replacement uses tints drawn from the existing project tokens — blue centre, red middle bands, mid-grey outer bands, darker red tails — to honour the epic-#309 "no new colour constants" rule and the dark-mode-safe palette contract documented at the top of `R/functions/main-functions.R`. The four bands are still visually distinct and each remains paired with its labelled percentage; reading the figure is unchanged.
- **No randomness** — These are analytic curves (`stats::dnorm`), so neither chunk needs `set.seed()`. The issue brief's suggested `set.seed(354)` is intentionally not used: there is no `rnorm()` call in either chunk.
