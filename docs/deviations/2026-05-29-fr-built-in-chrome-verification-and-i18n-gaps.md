## 2026-05-29 ‚Äî Verified Quarto built-in chrome localizes to French via `lang: fr`; logged reading-time filter as a remaining i18n gap

- **What** ‚Äî Built the FR target locally (`quarto render --profile fr`) and inspected the rendered `_book/fr/content-fr/welcome.html` HTML directly. Confirmed that `lang: fr` (set by `_quarto-fr.yml`, landed in PR [#399](https://github.com/lddurbin/twelve_days_to_deming/pull/399)) drives Quarto's bundled French translations for the entirety of the page chrome: `<html lang="fr">`, the page `<title>` ("Bienvenue ‚Äì 12 Days to Deming"), the sidebar toggle aria-labels ("Basculer la barre lat√©rale", "Basculer la section"), the search button aria-label ("Recherche"), and every search-result string (`"search-no-results-text": "Pas de r√©sultats"`, `"search-copy-link-title": "Copier le lien vers la recherche"`, etc.). No `_language-fr.yml` override was needed for any of these ‚Äî the gaps listed in #395's acceptance criteria as candidates for an override file were all already covered by Quarto's bundled localization at the version pin in CI (Quarto 1.9.37).
- **Where** ‚Äî No code changes. This file (`docs/deviations/2026-05-29-fr-built-in-chrome-verification-and-i18n-gaps.md`) is the only artifact from #395.
- **Source reference** ‚Äî n/a. Infrastructure verification; nothing in Neave's source materials is affected.
- **Why** ‚Äî #395 was scoped as a verification PR to confirm an architectural assumption from the multilingual ADR (#320): that `lang: fr` would do the heavy lifting for chrome localization and any project-specific override file would be small or empty. Confirming this lets downstream work (the switcher #322, pa11y parity #334) treat French chrome as "free" rather than budgeting for translation engineering.
- **Implementation note ‚Äî reading-time filter is not localized** ‚Äî [`filters/reading-time.lua`](../../filters/reading-time.lua) emits hardcoded English strings ("min reading", "min recommended", "+ activities") and English aria-labels ("Estimated reading time", "Estimated reading time and the author's recommended time including reflection and activities", etc.). The filter does not currently consult document or project `lang` metadata. This is not a Quarto-localizable gap (a Lua filter, not a built-in template), so no `_language-fr.yml` override applies. The gap is invisible on the current FR stub because the stub page is under the filter's `MIN_WORDS = 50` threshold ‚Äî the indicator block isn't emitted, so no English strings reach the page. Future FR pages with ‚â•50 words of prose will render the English strings until the filter is taught about `lang`.
- **Implementation note ‚Äî chapter-list concatenation discovered, filed as follow-up** ‚Äî During the verification render, `QUARTO_PROFILE=fr quarto inspect` showed that the resolved `project.render` list contains all 122 EN pages plus the one FR stub appended at position 94. Quarto's profile YAML merge does deep-merge with **list concatenation** on `book.chapters` and `book.appendices`, not replacement, which contradicts the design assumption in `_quarto-fr.yml` (#393) and the architectural decision (#320). The FR build is therefore currently producing a `_book/fr/` populated with EN content. This does not affect chrome verification ‚Äî the FR-specific page `content-fr/welcome.html` still renders with French chrome ‚Äî but it blocks #396's CI wiring. Filed as [#401](https://github.com/lddurbin/twelve_days_to_deming/issues/401) to land before #396.
- **Implementation note ‚Äî search aria-label JS patch in `_quarto.yml` is dormant under current Quarto pin** ‚Äî `_quarto.yml:218-222` contains a JS workaround that forces `aria-label="Search"` on `.quarto-search-button` when Quarto emits an empty `aria-label=""`. Inspection of the rendered FR HTML confirms Quarto 1.9.37 already emits `aria-label="Recherche"` server-side, so the `!s.getAttribute("aria-label")` guard fails and the English fallback never fires. The patch can be removed in a future cleanup PR once we're confident the pin won't regress, but it is not an active i18n gap today.
